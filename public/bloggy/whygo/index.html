I Love Go because of it's simplicity.


What does this mean? I am referring to its api and it's ability to uncouple code. Why is this important? Many developers are bound to the constraints to their language and tech stack. The point is so that when your project gets bigger you are not constrained to what you have already created.
An example of what I am referring to.

type Guitarist struct{Guitarist } //object
func(m Musician) Run(){} // method
type Musician interface { Run()} //interface that uses the method
The three bits of code that are stated are pretty simple. but they are the building blocks to uncoupling code.
  • 1. struct
  • 2. method
  • 3. interface
These are the components needed to uncouple your code. Lets say you are building a Service on Musican. 
func Service(Musician){
    fmt.Println(Musician)
}
We used the interface Musician to allow for any struct with run() to be passed. So if we needed to pass another struct of data in we can. This allows for not having to think about your data or concrete data type. Something many languages you are tied to. I will go further in the next blog about packages and utilizing this method. ...
